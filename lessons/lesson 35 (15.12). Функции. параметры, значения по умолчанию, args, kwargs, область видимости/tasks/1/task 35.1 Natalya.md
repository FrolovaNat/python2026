# Практическое задание 35.1

Это задание относится к первому блоку темы "Функции". Вы уже знакомы с коллекциями (списками, словарями) и базовыми алгоритмами. Сейчас мы начинаем изучать функции — основной инструмент для структурирования кода. В этом блоке вы научитесь объявлять функции с помощью `def`, передавать в них аргументы (параметры) и возвращать результат с помощью `return`. Это фундамент для всех последующих тем, связанных с функциями.

## Задания

### Описание задачи
Наталья, ваша задача — написать программу для простого анализа списка оценок студентов. Вам нужно создать функцию, которая принимает список целых чисел (оценки) и возвращает словарь с результатами анализа. Анализ включает:
1.  Средний балл (сумма всех оценок, деленная на их количество).
2.  Максимальную оценку.
3.  Минимальную оценку.

Функция должна **возвращать** словарь с ключами `"средний"`, `"максимум"`, `"минимум"` и соответствующими значениями (средний балл — число с плавающей точкой). Весь вывод (печать) должен происходить **за пределами** функции, в основном коде, после получения результата.

**Важно:** Используйте только пройденные темы: `def`, параметры, аргументы, `return`. Не используйте именованные аргументы, значения по умолчанию или другие конструкции, которые еще не изучались.

### Пример входных и выходных данных
**Пример 1:**
*   **Входные данные для функции:** Список `[5, 4, 3, 5, 2]`
*   **Что функция должна вернуть (словарь):**
    ```python
    {
        "средний": 3.8,
        "максимум": 5,
        "минимум": 2
    }
    ```
*   **Пример основного кода и вывода:**
    ```python
    # Вызываем функцию и сохраняем результат
    result = analyze_grades([5, 4, 3, 5, 2])
    # Печатаем результат
    print("Результат анализа:", result)
    ```
    **Вывод в консоли:**
    ```
    Результат анализа: {'средний': 3.8, 'максимум': 5, 'минимум': 2}
    ```

**Пример 2:**
*   **Входные данные для функции:** Список `[10, 9, 8]`
*   **Возвращаемый словарь:**
    ```python
    {
        "средний": 9.0,
        "максимум": 10,
        "минимум": 8
    }
    ```

### Критерии проверки и ограничения
1.  Функция должна быть объявлена с помощью `def` и иметь один параметр (например, `grades`).
2.  Внутри функции **не должно быть** вызовов `print()`. Все вычисленные значения должны собираться в словарь и возвращаться с помощью `return`.
3.  Функция должна корректно обрабатывать любой непустой список целых чисел.
4.  Для вычисления среднего балла используйте деление (`/`), результат должен быть числом с плавающей точкой.
5.  Для нахождения максимума и минимума используйте встроенные функции `max()` и `min()` или реализуйте линейный поиск, как изучали ранее.
6.  Основной код (после определения функции) должен вызывать ее с примером списка оценок, получать результат и выводить его на экран.

### Решение задачи (псевдокод)
1.  Объяви функцию с именем (например, `analyze_grades`), которая принимает один параметр — список.
2.  Внутри функции:
    a.  Вычисли сумму всех элементов списка.
    b.  Вычисли длину (количество элементов) списка.
    c.  Раздели сумму на длину, чтобы получить среднее значение. Сохрани результат.
    d.  Найди максимальный элемент в списке. Сохрани результат.
    e.  Найди минимальный элемент в списке. Сохрани результат.
    f.  Создай новый словарь.
    g.  Запиши в словарь по ключу `"средний"` вычисленное среднее значение.
    h.  Запиши в словарь по ключу `"максимум"` найденный максимум.
    i.  Запиши в словарь по ключу `"минимум"` найденный минимум.
    j.  Верни этот словарь как результат работы функции (используй `return`).
3.  В основном коде, после определения функции:
    a.  Создай тестовый список с оценками.
    b.  Вызови функцию, передав ей этот список в качестве аргумента, и сохрани возвращенный словарь в переменную.
    c.  Напечатай содержимое этой переменной.

### Рекомендации
-   Вынести в отдельную функцию: Всю логику анализа списка необходимо вынести в одну функцию, как и требуется в задании. Это хорошая практика — отделять код, который выполняет вычисления, от кода, который взаимодействует с пользователем (печатает).
-   PEP8:
    -   Используйте осмысленные имена для функции и параметра. Например, `analyze_grades` и `grades_list` лучше, чем `func1` и `lst`.
    -   После определения функции оставляйте две пустые строки перед началом основного кода.
