# Практическое задание 35.4

## Задания

### Описание задачи
В предыдущих блоках урока ты научилась создавать функции, передавать аргументы позиционно и по имени, использовать значения по умолчанию, а также работать с `*args` и `**kwargs`. Теперь важно закрепить понимание **области видимости переменных** — как функции взаимодействуют с внешним миром и какие переменные им доступны.

Твоя задача — написать программу, которая моделирует простой **счётчик активности** для нескольких пользователей. Программа должна уметь:
1.  **Создавать счётчик** для нового пользователя. При создании счётчик инициализируется нулём.
2.  **Увеличивать счётчик** активности конкретного пользователя.
3.  **Получать текущее значение** счётчика пользователя.
4.  **Сбрасывать счётчик** конкретного пользователя до нуля.

Все данные о пользователях и их счётчиках должны храниться в **глобальном словаре**. Твои функции будут работать с этим словарём, изменяя его содержимое. Это упражнение позволит на практике разобраться с локальными и глобальными переменными, а также с необходимостью использования ключевого слова `global`.

### Пример входных и выходных данных
**Пример последовательности вызовов функций:**
```python
# Изначально глобальный словарь пуст: user_counters = {}

create_counter("Наталья")  # Создаём счётчик для Натальи
print(get_counter("Наталья"))  # Получаем значение: должно быть 0

increment_counter("Наталья")  # Увеличиваем счётчик Натальи на 1
increment_counter("Наталья")  # Ещё раз увеличиваем
print(get_counter("Наталья"))  # Получаем значение: должно быть 2

create_counter("Иван")       # Создаём счётчик для Ивана
increment_counter("Иван")    # Увеличиваем счётчик Ивана
print(get_counter("Иван"))   # Получаем значение: должно быть 1
print(get_counter("Наталья")) # Значение Натальи осталось 2

reset_counter("Наталья")     # Сбрасываем счётчик Натальи
print(get_counter("Наталья")) # Получаем значение: должно быть 0

# Попытка получить счётчик несуществующего пользователя
print(get_counter("Мария"))  # Должно вернуть 0 (или сообщение об ошибке, в зависимости от реализации)
```

**Ожидаемый вывод в консоли:**
```
0
2
1
2
0
0
```

### Критерии проверки и ограничения
1.  **Глобальная переменная:** Для хранения данных должен использоваться один глобальный словарь с именем `user_counters`. Ключи — имена пользователей (строки), значения — целые числа (счётчики).
2.  **Функции:** Реализуй четыре функции, каждая из которых принимает имя пользователя (`username`) в качестве аргумента:
    *   `create_counter(username)`: Создаёт запись для пользователя в глобальном словаре со значением 0. Если запись уже существует, ничего не делает.
    *   `increment_counter(username)`: Увеличивает счётчик пользователя на 1. Если пользователя нет в словаре, сначала создаёт его запись (со значением 0), а затем увеличивает.
    *   `get_counter(username)`: Возвращает текущее значение счётчика пользователя. Если пользователя нет, возвращает 0.
    *   `reset_counter(username)`: Устанавливает счётчик пользователя в 0. Если пользователя нет, создаёт запись со значением 0.
3.  **Область видимости:** При изменении глобального словаря `user_counters` внутри функций необходимо явно указывать это с помощью ключевого слова `global`.
4.  **Запрещено:** Использовать темы, не входящие в блок 4 (область видимости, локальная/глобальная переменная, `global`, замыкание, интеграция). Нельзя использовать классы или другие сложные структуры, не изученные ранее.

### Решение задачи (псевдокод)
1.  Объяви глобальную переменную `user_counters` как пустой словарь.
2.  Функция `create_counter(username)`:
    *   Объяви, что будешь использовать глобальный словарь `user_counters`.
    *   Если `username` отсутствует среди ключей словаря `user_counters`:
        *   Добавь в словарь пару `username: 0`.
3.  Функция `increment_counter(username)`:
    *   Объяви, что будешь использовать глобальный словарь `user_counters`.
    *   Вызови функцию `create_counter(username)`, чтобы гарантировать наличие записи.
    *   Увеличь значение в словаре по ключу `username` на 1.
4.  Функция `get_counter(username)`:
    *   Верни значение из словаря `user_counters` по ключу `username`, если такой ключ есть.
    *   Если ключа нет, верни 0.
5.  Функция `reset_counter(username)`:
    *   Объяви, что будешь использовать глобальный словарь `user_counters`.
    *   Установи значение в словаре по ключу `username` равным 0 (это создаст запись, если её не было).

### Рекомендации
-   **Вынести в отдельную функцию:** В данном задании вся логика уже разбита на четыре чёткие функции. Дополнительное выделение функций не требуется. Обрати внимание, как функция `increment_counter` переиспользует `create_counter` для обеспечения наличия пользователя в словаре — это хороший пример интеграции.
-   **PEP8:**
    -   Используй осмысленные имена функций, которые отражают их действие (`create_counter`, `increment_counter`).
    -   После объявления глобальной переменной `user_counters` оставь две пустые строки перед определением функций, чтобы визуально отделить данные от кода.
