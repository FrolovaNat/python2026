# Практическое задание 35.2

В предыдущем блоке вы научились объявлять функции, передавать аргументы и возвращать результат. Сейчас мы переходим к управлению аргументами: использованию значений по умолчанию и правильному порядку их передачи. Это поможет избежать частых ошибок, особенно при работе с изменяемыми типами данных, и сделает ваши функции более гибкими и безопасными.

## Задания

### Описание задачи
Наталья, вы помогаете разрабатывать систему для управления списком задач в небольшом проекте. Нужно создать функцию `add_task`, которая добавляет новую задачу в общий список. У каждой задачи есть обязательное название (`title`) и опциональные параметры: приоритет (`priority`) и список тегов (`tags`).

Функция должна работать по следующим правилам:
1.  Параметр `title` — обязательный, позиционный.
2.  Параметр `priority` — необязательный, именованный, со значением по умолчанию `"medium"`. Допустимые значения: `"low"`, `"medium"`, `"high"`.
3.  Параметр `tags` — необязательный, именованный, со значением по умолчанию. Это должен быть список строк. Важно избежать классической ошибки с изменяемым значением по умолчанию.
4.  Функция возвращает словарь, представляющий добавленную задачу, например: `{"title": "Проверить код", "priority": "high", "tags": ["bugfix", "urgent"]}`.
5.  Общий список задач (`task_list`) передается в функцию как первый аргумент. Это изменяемый список, в который функция добавляет созданный словарь задачи.

Также создайте функцию `get_high_priority_tasks`, которая принимает список задач и возвращает новый список, содержащий только задачи с приоритетом `"high"`. Параметр для списка задач должен иметь значение по умолчанию — пустой список, созданный безопасным способом.

### Пример входных и выходных данных
**Пример 1: Добавление задачи с разными аргументами**
```python
my_tasks = []

# 1. Только обязательный аргумент
task1 = add_task(my_tasks, "Написать отчет")
print(task1)
# Вывод: {'title': 'Написать отчет', 'priority': 'medium', 'tags': []}
print(my_tasks)
# Вывод: [{'title': 'Написать отчет', 'priority': 'medium', 'tags': []}]

# 2. С указанием приоритета (именованный аргумент)
task2 = add_task(my_tasks, "Позвонить клиенту", priority="high")
print(task2)
# Вывод: {'title': 'Позвонить клиенту', 'priority': 'high', 'tags': []}

# 3. С указанием тегов (именованный аргумент, изменяемый тип)
task3 = add_task(my_tasks, "Исправить баг", tags=["bugfix", "urgent"])
print(task3)
# Вывод: {'title': 'Исправить баг', 'priority': 'medium', 'tags': ['bugfix', 'urgent']}

# 4. Все аргументы, порядок именованных не важен
task4 = add_task(my_tasks, "Обновить документацию", tags=["docs"], priority="low")
print(task4)
# Вывод: {'title': 'Обновить документацию', 'priority': 'low', 'tags': ['docs']}

print(len(my_tasks))  # Вывод: 4 (все задачи добавлены в один список)
```

**Пример 2: Поиск срочных задач**
```python
# Используем список my_tasks из предыдущего примера
urgent_tasks = get_high_priority_tasks(my_tasks)
print(urgent_tasks)
# Вывод: [{'title': 'Позвонить клиенту', 'priority': 'high', 'tags': []}]

# Вызов с пустым списком по умолчанию
empty_urgent = get_high_priority_tasks()
print(empty_urgent)  # Вывод: []
```

### Критерии проверки и ограничения
- Функция `add_task` строго принимает аргументы в порядке: `task_list`, `title`, `priority=...`, `tags=...`.
- Значение по умолчанию для параметра `tags` должно быть задано безопасно, чтобы списки тегов разных задач не смешивались.
- Функция `get_high_priority_tasks` должна корректно обрабатывать вызов без аргументов, возвращая пустой список.
- В решении нельзя использовать `*args`, `**kwargs`, распаковку, глобальные переменные (кроме области видимости функций) или другие темы из будущих блоков.
- Код должен соответствовать PEP8 (отступы 4 пробела, пробелы вокруг операторов, осмысленные имена переменных).

### Решение задачи (псевдокод)
**Для функции add_task:**
1.  Объявить функцию с параметрами: task_list, title, priority, tags.
2.  Для параметра priority установить значение по умолчанию "medium".
3.  Для параметра tags установить безопасное значение по умолчанию (использовать None-паттерн).
4.  Внутри функции:
    a. Если tags равно None, создать новый пустой список.
    b. Создать словарь задачи с ключами "title", "priority", "tags" и соответствующими значениями.
    c. Добавить этот словарь в конец списка task_list.
    d. Вернуть созданный словарь.

**Для функции get_high_priority_tasks:**
1.  Объявить функцию с одним параметром task_list.
2.  Установить для task_list безопасное значение по умолчанию (использовать None-паттерн).
3.  Внутри функции:
    a. Если task_list равно None, создать новый пустой список.
    b. Создать новый пустой список для результата.
    c. Для каждого элемента (словаря задачи) в task_list:
        - Если значение по ключу "priority" равно строке "high", добавить этот словарь в список результата.
    d. Вернуть список результата.

### Рекомендации
- Вынести в отдельную функцию: Сами функции `add_task` и `get_high_priority_tasks` уже являются логическими единицами. Дополнительного вынесения не требуется.
- PEP8:
    - Используйте осмысленные имена переменных: `task_list` вместо `lst`, `high_priority_tasks` вместо `result`.
    - После запятой в параметрах функции и элементах списка ставьте пробел: `def func(a, b, c=1):`, `tags=["bugfix", "urgent"]`.
