# Домашняя работа: Функции. Параметры, значения по умолчанию, args, kwargs, область видимости

## Задачи
1. **Функция для форматирования данных о товаре**
   - **Описание задачи:** Тренировка объявления функций, передачи позиционных и именованных аргументов, использования `return`. Задача развивает навык разделения логики вычисления и вывода результата, что важно для ответственного подхода к структуре кода.
   - **Пример (вход → выход):**
     - `format_product("Молоко", 90)` → `"Товар: Молоко, Цена: 90 руб."`
     - `format_product("Хлеб", price=50)` → `"Товар: Хлеб, Цена: 50 руб."`
     - `format_product(price=120, name="Яблоки")` → `"Товар: Яблоки, Цена: 120 руб."`
   - **Критерии проверки и ограничения:**
     - Функция должна принимать два параметра: `name` и `price`.
     - Функция должна возвращать строку в указанном формате, а не печатать ее.
     - Должна корректно работать как с позиционной, так и с именованной передачей аргументов.
   - **Решение (псевдокод):**
     1. Объявить функцию с двумя параметрами: `name` и `price`.
     2. Внутри функции создать строку результата, подставив значения параметров.
     3. Вернуть эту строку с помощью оператора `return`.
   - **Рекомендации:**
     - Используйте f-строки для формирования результата.
     - Следуйте PEP8: имена функций в `snake_case`, после объявления функции оставьте две пустые строки.

2. **Безопасное добавление заметок с датой**
   - **Описание задачи:** Закрепление работы со значениями по умолчанию, в частности, с паттерном `None` для избежания ловушки изменяемых объектов. Упражнение требует внимательности к деталям, что соответствует ответственному подходу студента.
   - **Пример (вход → выход):**
     - `add_note("Купить молоко")` → `['2024-12-15: Купить молоко']`
     - `add_note("Позвонить маме")` → `['2024-12-15: Позвонить маме']`
     - `my_list = []; add_note("Первая", my_list); add_note("Вторая", my_list)` → `['2024-12-15: Первая', '2024-12-15: Вторая']`
   - **Критерии проверки и ограничения:**
     - Функция должна иметь два параметра: `text` и `notes_list` (со значением по умолчанию `None`).
     - Если `notes_list` равен `None`, внутри функции нужно создать новый пустой список.
     - К тексту заметки нужно добавить текущую дату в формате "ГГГГ-ММ-ДД:" (можно использовать фиктивную дату, например, "2024-12-15").
     - Функция должна возвращать обновленный список.
   - **Решение (псевдокод):**
     1. Объявить функцию с параметрами `text` и `notes_list` (по умолчанию `None`).
     2. Если `notes_list` равен `None`, создать новый пустой список.
     3. Сформировать строку заметки: дата + текст.
     4. Добавить эту строку в список `notes_list`.
     5. Вернуть список `notes_list`.
   - **Рекомендации:**
     - Для проверки на `None` используйте оператор `is`.
     - Следуйте PEP8: после запятой в параметрах ставьте пробел.

3. **Универсальный калькулятор счета**
   - **Описание задачи:** Применение `*args` для работы с переменным числом аргументов и значений по умолчанию. Задача развивает абстрактное мышление, требуя создать гибкую функцию, которая может обрабатывать разное количество данных.
   - **Пример (вход → выход):**
     - `calculate_total(100, 200, 300)` → `600`
     - `calculate_total(50, 75, tip_percent=10)` → `137.5` (Сумма 125 + 10% чаевых)
     - `calculate_total(400, discount=50)` → `350` (Сумма 400 - скидка 50)
   - **Критерии проверки и ограничения:**
     - Функция должна принимать:
       - `*args` для позиционных аргументов (цены товаров).
       - `tip_percent` (процент чаевых) со значением по умолчанию `0`.
       - `discount` (фиксированная скидка) со значением по умолчанию `0`.
     - Сначала считается сумма всех цен из `args`.
     - Затем к сумме применяется процент чаевых.
     - Затем из результата вычитается фиксированная скидка.
     - Функция должна возвращать итоговое число.
   - **Решение (псевдокод):**
     1. Объявить функцию с параметрами `*args`, `tip_percent=0`, `discount=0`.
     2. Вычислить сумму всех элементов в `args`.
     3. Рассчитать чаевые: сумма * (`tip_percent` / 100).
     4. Прибавить чаевые к сумме.
     5. Вычесть `discount` из результата.
     6. Вернуть итоговое значение.
   - **Рекомендации:**
     - Для суммирования `args` используйте встроенную функцию `sum()`.
     - Убедитесь, что функция возвращает число, даже если `args` пуст (вернется отрицательное значение скидки, что корректно для граничного случая).

4. **Обработка профиля пользователя с распаковкой**
   - **Описание задачи:** Интеграция концепций `**kwargs`, распаковки словаря и области видимости. Сложность заключается в комбинировании изученных элементов в одном сценарии, что требует системного мышления.
   - **Пример (вход → выход):**
     ```python
     base_info = {"name": "Наталья", "age": 25}
     extra_info = {"city": "Москва", "language": "Python"}
     result = create_profile(**base_info, **extra_info)
     # result должен быть: {'name': 'Наталья', 'age': 25, 'city': 'Москва', 'language': 'Python', 'level': 'Новичок'}
     ```
   - **Критерии проверки и ограничения:**
     - Функция `create_profile` должна принимать произвольные именованные аргументы (использовать `**kwargs`).
     - Внутри функции создайте локальную переменную `level` со значением "Новичок".
     - Функция должна возвращать словарь, содержащий все переданные аргументы и добавленное поле `"level"`.
     - В основном коде продемонстрируйте вызов функции с распаковкой двух словарей (`**base_info, **extra_info`).
   - **Решение (псевдокод):**
     1. Объявить функцию с параметром `**kwargs`.
     2. Внутри функции создать локальную переменную `level` = "Новичок".
     3. Создать новый словарь `profile`.
     4. Добавить в `profile` все пары ключ-значение из `kwargs`.
     5. Добавить в `profile` пару `"level": level`.
     6. Вернуть словарь `profile`.
     7. В основном коде создать два словаря с данными.
     8. Вызвать функцию, распаковав оба словаря в ее аргументы.
   - **Рекомендации:**
     - Для создания итогового словаря можно сначала скопировать `kwargs` (`result = kwargs.copy()`), а затем добавить ключ `'level'`.
     - Следуйте PEP8: ограничьте длину строки вызова функции с распаковкой (можно перенести аргументы на новую строку).
