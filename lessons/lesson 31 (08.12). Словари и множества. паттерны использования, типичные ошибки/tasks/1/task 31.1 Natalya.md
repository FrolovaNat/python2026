# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Хранилище музея помечает, в какой комнате находится каждый экспонат. Куратор сверяет список перед ревизией. Создай словарь artifact_room, где ключ — инвентарный номер, а значение — комнату хранения. Затем выведи информацию для переменных to_move и lost_item.

## Пример входных данных
```python
artifacts = [
    ('mask-17', 'Bunker-2'),
    ('vase-03', 'Vault-1'),
    ('painting-12', 'Hall-5'),
]
to_move = 'vase-03'
lost_item = 'lamp-09'
```

## Пример выходных данных
```text
vase-03 -> Vault-1
lamp-09 -> не найден
```

## Критерии успешного выполнения
- Список пар преобразован в словарь artifact_room с ключом 'инвентарный номер'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для to_move и lost_item в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку artifacts в цикле и наполняй словарь artifact_room.
- Используй множественное присваивание: key, value = запись из artifacts.
- Для отсутствующих данных возвращай строку 'не найден'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
