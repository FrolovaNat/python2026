# Вопрос 1. Тема: Основы SQL
  
**1. Какая команда используется для извлечения всех записей из таблицы в SQL?**
  
Варианты ответов:
1) SELECT 1 FROM table_name;
2) ✅ SELECT * FROM table_name;
3) SELECT all FROM table_name;
4) SELECT FROM table_name;
  
### ИИ анализ
Команда `SELECT * FROM table_name;` является стандартным синтаксисом SQL для извлечения всех столбцов и всех записей из указанной таблицы.  Символ `*` является специальным оператором, обозначающим "все столбцы".  Другие варианты либо извлекают конкретное значение, используют несуществующий синтаксис, либо являются неполными командами.  

# Вопрос 2. Тема: Основы SQL
  
**2. Как ограничить выборку по конкретному условию?**
  
Варианты ответов:
1) ✅ SELECT * FROM table_name WHERE condition;
2) SELECT * WHERE condition FROM table_name;
3) SELECT * FROM table_name ORDER BY condition;
4) SELECT WHERE condition FROM table_name;
  
### ИИ анализ
Для фильтрации записей по условию в SQL используется предложение `WHERE`.  Правильный синтаксис требует, чтобы `WHERE` следовало после `FROM table_name`.  Вариант 2 нарушает порядок предложений, вариант 3 использует `ORDER BY` для сортировки, а не фильтрации, а вариант 4 пропускает обязательное ключевое слово `SELECT *`.  

# Вопрос 3. Тема: Основы SQL
  
**3. Как отсортировать результаты по убыванию значения в столбце age?**
  
Варианты ответов:
1) SELECT * ORDER BY age FROM table_name;
2) ✅ SELECT * FROM table_name ORDER BY age DESC;
3) SELECT * FROM table_name ORDER BY DESC age;
4) SELECT * FROM table_name WHERE age ORDER BY DESC;
  
### ИИ анализ
Для сортировки по убыванию используется ключевое слово `DESC` после имени столбца в предложении `ORDER BY`.  Правильный порядок предложений: `SELECT ... FROM ... ORDER BY ...`.  В варианте 1 нарушен порядок (`ORDER BY` до `FROM`), в варианте 3 неверный порядок слов (`DESC` должно быть после имени столбца), а вариант 4 пытается использовать `WHERE` без условия.  

# Вопрос 4. Тема: Основы SQL
  
**4. Как выбрать записи, где age меньше 30 или salary больше 50000, и отсортировать их по name?**
  
Варианты ответов:
1) SELECT * FROM table_name WHERE age < 30 OR salary > 50000 ORDER BY name;
2) ❌ SELECT * FROM table_name WHERE age < 30 AND salary > 50000 ORDER BY name;
3) SELECT * FROM table_name ORDER BY age < 30 OR salary > 50000, name;
4) SELECT * FROM table_name WHERE age < 30 OR salary > 50000 name;
  
### ИИ анализ
Вы выбрали вариант 2, который использует логический оператор `AND`.  Это означает, что будут выбраны записи, где **одновременно** `age < 30` И `salary > 50000`, что не соответствует условию задачи "или".  Правильным ответом является вариант 1, где используется оператор `OR` для выбора записей, удовлетворяющих хотя бы одному из условий, и корректно применяется `ORDER BY name` для сортировки.  

# Вопрос 5. Тема: Основы SQL
  
**5. Как выбрать записи, где salary больше 50000 и age больше 25?**
  
Варианты ответов:
1) SELECT * FROM table_name WHERE salary > 30000 NOT age > 25;
2) SELECT * FROM table_name WHERE salary > 30000, age > 25;
3) SELECT * FROM table_name WHERE salary > 30000 OR age > 25;
4) ✅ SELECT * FROM table_name WHERE salary > 30000 AND age > 25;
  
### ИИ анализ
Для одновременного выполнения двух условий в SQL используется логический оператор `AND`.  Вариант 4 корректно применяет его, требуя, чтобы оба условия (`salary > 30000` и `age > 25`) были истинными.  Вариант 1 использует неверный синтаксис с `NOT`, вариант 2 применяет недопустимый разделитель-запятую, а вариант 3 использует `OR`, что соответствует выбору по любому из условий, а не по обоим сразу.
