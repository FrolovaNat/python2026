# Вопрос 1. Блок 5. Box model и управление отступами
  
### Содержание и вопросы

- Что входит в box model?
- Чем `margin` отличается от `padding`?
- Зачем нужен `box-sizing: border-box`?
- Почему элемент “не влезает” в ширину?

### Материал (лаконично)

- У каждого блока есть “коробка”.
- `padding` это воздух внутри блока. Он отодвигает текст от края рамки.
- `margin` это расстояние снаружи блока. Он раздвигает блоки друг от друга.
- Если ты задаешь элементу ширину и добавляешь padding и рамку, без `border-box` элемент может стать шире, чем ты ожидал.
- `box-sizing: border-box` делает ширину предсказуемой. Padding и рамка входят в заданную ширину.

#  Вопрос 2. Практическое задание (Блок 5)
  
### Условия

Наведи порядок в отступах пошагово:

1. В `styles.css` включи `border-box` для всех элементов одним правилом.
2. Выбери один основной вертикальный отступ и используй его для секций.
3. Применяй этот отступ ко всем секциям одинаково.
4. Для карточек проектов:
   - добавь внутренний отступ
   - добавь рамку или фон
   - добавь скругление углов
5. Открой страницу и проверь:
   - текст не прилипает к краям
   - расстояния между секциями выглядят одинаково

Ограничения:

- Отступы должны быть согласованными. Не делай “случайные” значения для каждого элемента.

### Псевдокод решения

```text
IN CSS_FILE
  APPLY BORDER_BOX_TO_ALL

  FOR EACH SECTION_BLOCK
    SET OUTER_SPACING

  FOR EACH CARD_BLOCK
    SET INNER_SPACING
    SET BORDER_OR_BG
    SET CORNER_ROUNDING
END
```

### Если использовал ИИ, обязательные изменения вручную

- Перестрой отступы так, чтобы они задавались через 1-2 “основных” правила, а не через множество точечных.
- Добавь правило, которое улучшает поведение изображений в карточках (без фиксированной ширины).
