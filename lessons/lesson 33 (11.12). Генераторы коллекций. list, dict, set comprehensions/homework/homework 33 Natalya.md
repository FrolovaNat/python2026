# Домашнее задание. Урок 33. Генераторы коллекций. list, dict, set comprehensions

## Краткая сводка
- **Проблемные блоки:** неверное форматирование строк в list comprehension, игнорирование условий фильтрации, синтаксические ошибки при создании dict comprehension.
- **Сильные стороны:** умеешь выбирать элементы по длине строки, используешь comprehension как инструмент.
- **Приоритеты:** закрепить корректный вывод с тире и падежами, научиться записывать фильтры прямо в comprehension, аккуратно строить dict comprehension с тернарными выражениями.

## Задача 1. Повторение сложной темы — «короткие подписи сотрудников»
- **Блок:** list comprehension + форматирование.
- **Описание:** дан список имён `names`. Построй список строк `"Имя — N букв/буквы"` только для имён длиной < 6. Используй:
  - длинное тире с пробелами ` — `;
  - слово «буквы» для длины 2–4 и «букв» для остальных;
  - исходный порядок не меняй.
- **Пример входа:** `["Антон", "Екатерина", "Ира", "Юля"]`
- **Ожидаемый вывод:** `["Антон — 5 букв", "Ира — 3 буквы", "Юля — 3 буквы"]`
- **Критерии:** один list comprehension, корректное склонение, исходный список не изменяется.

## Задача 2. Закрепление практики — «компактные встречи»
- **Блок:** list comprehension с несколькими условиями.
- **Описание:** есть список кортежей `meetings = [(topic, members, minutes), ...]`. Собери список строк `"<topic>: <members> чел, <minutes> мин"` только если `members < 5` **и** `minutes <= 30`.
- **Граничные случаи:** значения уже целые; не выводи ничего, если условия не выполняются.
- **Критерии:** один comprehension, оба условия объединены в конце, никаких `print` внутри, результат сохраняется в новую переменную `compact_meetings`.

## Задача 3. Интеграция блоков — «статус городских митапов»
- **Блок:** dict comprehension + тернарный оператор.
- **Описание:** список словарей `meetups = [{"city": "...", "tickets": 80, "sold": 75}, ...]`. Создай словарь `availability`, где ключ — город, значение — строка:
  - `"распродан"` если `sold >= tickets`;
  - `"остались места"` иначе.
- Включай только города, где `tickets >= 50`.
- **Критерии:** один dict comprehension, строка выбирается через тернарный оператор, условие по `tickets` находится после comprehension, никаких синтаксических ошибок.

## Задача 4. Подготовка к следующему занятию — «фильтр активностей»
- **Блок:** сочетание set/list comprehension с проверкой входных данных (подготовка к темам о структурах данных).
- **Описание:** есть список строк `activity_logs = ["support:call:12", "it:deploy:45", ...]`.
  1. Построй множество `fast_departments` (set comprehension) из департаментов, где действие `"call"` или `"review"` и длительность < 20.
  2. Одновременно создай список `slow_entries` (list comprehension) строк `"dept -> action (duration)"` для записей с длительностью ≥ 20.
- **Требования:** каждая строка должна состоять из трёх сегментов разделённых `":"`; пропусти строки с другим форматом.
- **Критерии:** два отдельные comprehension без вспомогательных циклов, проверка формата, единое преобразование `int(duration)` внутри каждого выражения.
