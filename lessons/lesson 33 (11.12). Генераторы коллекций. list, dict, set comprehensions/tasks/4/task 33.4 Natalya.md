# Задание 33.4. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«Set comprehensions» помогает быстро получить множество уникальных значений после фильтрации данных.

## Условие задачи
**Сложность**: <span style="color: #f9a825">● средний</span>. примерное время: 15–20 минут

Есть список `bookings`, где каждый элемент — словарь `{ "city": str, "status": str }`. Построй множество `open_cities`, включающее города, где статус `"confirmed"` или `"paid"`. Названия городов должны быть приведены к нижнему регистру. Используй одно выражение set comprehension.

## Пример входных данных
```
bookings = [
    {"city": "Moscow", "status": "confirmed"},
    {"city": "Kazan", "status": "draft"},
    {"city": "Sochi", "status": "paid"},
    {"city": "Perm", "status": "canceled"}
]
```

## Пример выходных данных
```
{'moscow', 'sochi'}
```

## Критерии успешного выполнения
- Множество `open_cities` формируется одним set comprehension.
- В него попадают только города со статусами `confirmed` или `paid`.
- Названия городов приводятся к нижнему регистру.
- Исходный список `bookings` не изменяется.

## Подсказки
- Фильтр удобно записать как `if booking["status"] in {"confirmed", "paid"}`.
- Приводить к нижнему регистру лучше после фильтрации, чтобы не тратить операции впустую.

## Частые ошибки и ограничения
- Не оборачивай comprehension в `set()` и не используй `add`.
- Следи, чтобы в списках не было больше пяти пунктов.
- Не меняй исходные словари.
