# Задание 33.3. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«Dict comprehensions» помогает быстро получить словарь с новыми значениями, сохраняя читаемость кода.

## Условие задачи
**Сложность**: <span style="color: #f9a825">● средний</span>. примерное время: 15–20 минут

Есть список `meetups`, где каждый элемент — словарь вида `{ "city": str, "tickets": int, "sold": int }`. Построй словарь `availability`, в котором ключ — название города, а значение — строка `"распродан"`, если проданных билетов не меньше `tickets`, иначе `"остались места"`. Добавляй в результат только те города, у которых общее количество билетов не меньше 50.

## Пример входных данных
```
meetups = [
    {"city": "Москва", "tickets": 80, "sold": 80},
    {"city": "Казань", "tickets": 45, "sold": 30},
    {"city": "Пермь", "tickets": 60, "sold": 40}
]
```

## Пример выходных данных
```
{'Москва': 'распродан', 'Пермь': 'остались места'}
```

## Критерии успешного выполнения
- Словарь `availability` создаётся одним dict comprehension.
- В словарь попадают только города с `tickets >= 50`.
- Статус выбирается условием внутри comprehension.
- Исходный список `meetups` остаётся неизменным.

## Подсказки
- Перебирай элементы списка так: `for meetup in meetups`.
- Выражение для значения можно записать через тернарный оператор.
- Фильтр по количеству билетов ставь в конце comprehension.

## Частые ошибки и ограничения
- Не используй внешние циклы или `dict()` поверх comprehension.
- Следи, чтобы ключи словаря не повторялись.
- Не добавляй больше пяти пунктов в один список.
